初识 Nginx

Nginx 是一个高性能的 HTTP 和反向代理服务器，也是一个 IMAP/POP3/SMTP 代理服务器。由于其高效的处理能力和轻量级的设计，Nginx 成为了互联网应用中广泛使用的 Web 服务器之一。

在我开始接触 Nginx 时，主要是为了提升 Web 服务的性能和处理能力。Nginx 作为反向代理服务器能够帮助我实现负载均衡、流量分发等功能，而作为 Web 服务器，它也能处理静态文件和提供动态内容。

配置反向代理（可以解决跨域问题）

Nginx 作为反向代理服务器，可以将客户端的请求转发到其他应用服务器。以下是一个简单的反向代理配置：

配置多个API请求

http {
    server {
        listen 80;
        server_name api.example.com;
    # 请求 /user 开头的路径转发到用户服务
    location /user/ {
        proxy_pass http://localhost:3001/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    # 请求 /order 开头的路径转发到订单服务
    location /order/ {
        proxy_pass http://localhost:3002/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    # 请求 /product 开头的路径转发到商品服务
    location /product/ {
        proxy_pass http://localhost:3003/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}

}

💡 注意事项：

  1. proxy_pass 后面的地址末尾 / 是否保留，影响路径拼接规则：
    • 有 /（如：proxy_pass http://localhost:3001/;）：只替换前缀，不保留原始路径。
    • 无 /（如：proxy_pass http://localhost:3001;）：会把原始路径附加到目标地址后。
